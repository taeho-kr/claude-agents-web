# Agent Common Rules

> 모든 에이전트가 공유하는 규칙. 개별 에이전트 프롬프트에 포함하지 않습니다.
> 오케스트레이터가 에이전트 호출 시 Persistent Context와 함께 자동 주입합니다.

---

## 코드 품질

- 함수 50줄, 파일 500줄 기준 초과 금지
- PoC라는 이유로 낮은 품질 코드 금지
- 유지보수 가능한 구조화된 코드만 작성
- 설계 없이 구현 시작 금지

## 모호성 처리

- 명세가 불명확하면 추측하지 않고 결과물에 "미결 사항" 섹션으로 기록
- 구현 에이전트: 불명확한 부분은 구현하지 않고 TODO 주석 + 미결 사항 보고

## 산출물 규칙

- 산출물은 반드시 `references/output-contracts.md`에 정의된 포맷을 따름
- .omc/ 파일 작성 시 필수 섹션 누락 금지
- 결과물의 첫 줄에 요약(PASS/FAIL, 수량 등) 포함

## 보안

- 하드코딩된 시크릿, API 키 금지
- 사용자 입력은 항상 검증 후 사용
- SQL 인젝션, XSS, CSRF 방지 기본 적용

## 협업

- 이전 에이전트의 산출물(.omc/)을 반드시 읽고 작업 시작
- 작업 완료 후 자신의 산출물을 .omc/에 기록
- 다른 에이전트의 산출물을 덮어쓰지 않음 (append 또는 별도 파일)
